# clbs-youtube-script-pro

顔出しアバター動画（HeyGen等）の**企画考案 → 台本執筆**を一気通貫で行い、後段の **clbs-youtube-edit**（編集スキル）にそのまま投入できる `script.txt` / `heygen_script.txt` / `media_list.yaml` を出力する。

YouTubeアルゴリズムに沿って「視聴者が思わずクリックし、最後まで見て、行動する」企画を考え抜くことを最優先する。台本の上手さより**企画の強さ**が再生数とCVRを決める。

## 出力ファイル（プロジェクトフォルダ `videos/YYYYMMDD_<title>/`）

| ファイル | 内容 |
|---|---|
| `script.txt` | 演出タグ＋breakを全て含む完成台本（編集スキルの正） |
| `heygen_script.txt` | `[…]`視覚・見出しタグを除去し `<break>` のみ残した版（HeyGen貼り付け用） |
| `media_list.yaml` | 素材台帳（headings / brolls / pictures / slides / thumbnail） |

# ワークフロー（7フェーズ・各フェーズで承認ゲート）

各フェーズで必ず立ち止まりユーザー承認を取る。一度に全部出力しない。ヒアリング済み情報があればスキップ可。

```
0 ヒアリング
1 軽量リサーチ（任意）
2 ジャンル適性チェック
3 企画考案：精鋭3案＋審査5項目セルフ採点   ← 本丸
4 構成案＋冒頭30秒フック設計
5 台本執筆（パートごと承認）
6 素材台帳出力
```

## フェーズ0：ヒアリング

企画の精度は素材で決まる。次を確認する（既出ならスキップ）:

- 職業・肩書き／ターゲット（誰のどんな悩み）／ゴール（誘導先＝LINE・登録・来店等）／話し方トーン
- **競合チャンネル**（あれば。フェーズ1リサーチの起点）
- **視聴者に持ってほしい「新しい常識」** と **今ターゲットが抱えている「間違った思い込み」**
- **絶対に伝えたいノウハウ・独自エピソード**（⚠️ これとユーザー提出資料**だけ**を台本の中身に使う。ナレッジの事例は流用しない＝コンテンツ汚染防止）
- **戦略タイプ**：検索・関連型（資産型）／ブラウジング型（バズ型）／両方／未定

## フェーズ1：軽量リサーチ（任意・WebSearch内蔵）

競合URLやジャンルが分かる場合のみ実行。重い分析はせず要点だけ掴む。本格的な異常値分析・バズ方程式モデリングが必要なら別スキル `clbs-youtube-research` を案内する。

手順:
1. WebSearchで「競合チャンネル名 人気動画」「ジャンル名 YouTube 急上昇/伸びてる」等を検索
2. 次の3点だけ抽出して会話に提示:
   - **勝ちパターン**：競合で当たっているタイトルの型（1〜3個）
   - **gap（空き）**：競合がまだやっていない切り口（最重要・3つ）
   - **推奨戦略**：検索型/ブラウジング型のどちらが有利か
→ 承認。スキップ時はフェーズ2へ。

## フェーズ2：ジャンル適性チェック

企画を出す前に視聴者パイを確認する。**ニッチすぎると構造的に伸びない。**

```
□ A：ジャンルのパイは十分広いか？
  ✅ 美容・健康・ダイエット・お金・副業・起業・人間関係・育児（何百万人が悩む）
  ❌ 特定の職種・業界の人しか関心を持たない
□ B：ターゲットが「ジャンルを知る前から悩んでいる」か？
  ✅ 「肩こりが治らない」「老後のお金が不安」（専門知識ゼロでも自分ごと）
  ❌ 「◯◯メソッドを使った集客法」（専門用語を知る人しかピンとこない）
□ C：「どう稼ぐか/売るか」が動画の中心になっていないか？
  ✅ 視聴者の生活の悩み・願望が中心
  ❌ 「自分のビジネスをどう伸ばすか」が中心（視聴者がオーナー限定に）
```
NGなら「生活上の悩み起点」へポジショニング拡張を提案してから次へ。

## フェーズ3：企画考案 — 精鋭3案（本丸）

10案乱発はしない。**考え抜いた3案**を出す。各案は次を必ず内蔵する。

各案の構成要素:
- **常識破壊フレームを核に**（下記8フレームから選ぶ）
- **心理トリガーを補助**（下記バズ10軸から1〜2個）
- **戦略タイプ別の設計**：検索型=検索意図＋キーワード／ブラウジング型=感情トリガー＋話題性
- **専門用語を日常語に変換**（NG「共鳴場理論」→OK「脳が勝手に変わる習慣」）
- **メカニズムに固有名詞**（「神経パズル」「量子アンテナ」型のチャンネル独自ワード）
- **タイトル2〜3バリエーション**（パワーワードを使う）
- **サムネ文字要素案**（10〜15字）

出力フォーマット（各案）:
```
【企画案N】
  タイトル案（2〜3バリエーション）
  戦略タイプ：検索型 / ブラウジング型
  常識破壊フレーム：（番号と名前）
  補助の心理トリガー：（バズ10軸のどれか）
  狙い：どの間違った常識をどう覆すか
  自己投影：視聴者が「これ私だ」と思う理由
  メカニズム固有名詞：
  サムネ文字案（10-15字）：
  ★企画審査5項目セルフ採点：1○ 2○ 3△ 4○ 5○（4/5 合格）
```
**5項目中4以上で合格。満たない案は作り直してから提示する。** → 3案提示し1案選択。

## フェーズ4：構成案＋冒頭30秒フック設計

決定企画で骨子を作る（本文はまだ書かない）。サムネ案も同時提示。

構成（YouTube台本テンプレート準拠）:
```
■ OP（0-2分）冒頭30秒フックを明示設計
   - テーマ＋衝撃的な結論（日常会話で）
   - 天国と地獄：実践後のベネフィット ⇔ 知らないままの損失
   - 希少性・権威性（なぜあなたが語れるか）
■ 共感（2-4分）：悩み代弁＋あるある（プロスペクト）＋yesセット3〜4連
■ 解決策：解決後の未来＋本題前CTA（LINE等）
■ 本題：3〜4見出し（ユーザーのノウハウ・資料に基づく解説。各見出しでアハ体験を作る）
■ まとめ：総括＋最終CTA
```
→ 承認。

## フェーズ5：台本執筆（パートごと承認）

1パートずつ執筆して提示。各パート冒頭に `[見出し：◯◯]`、論理パートに `[スライドN]`、感情・具体物に `[ピクチャーN]`、没入演出に `[BロールN]`＋`[カムリターン]`。全文末・全タグ直後に `<break time="0.8s" />`。冒頭OPにはフェーズ4のフック設計を反映。→ パートごと承認。

執筆ルールは後述の「執筆ルール」「タグ仕様」に従う。

## フェーズ6：素材台帳出力

`script.txt`（完成版）→ `heygen_script.txt`（タグ除去版）→ `media_list.yaml`（headings/brolls/pictures/slides/thumbnail）。番号整合性チェック・完了チェックリストを通す。

# 企画考案リファレンス（自己完結・外部資料不要）

## 常識破壊8フレーム（企画3案の核に使う）

| # | フレーム | 定型文の例 |
|---|---|---|
| 1 | 実は"逆"だった | 「実は○○な人ほど、××をしていない」「多くの人がやってる○○、実は逆効果」 |
| 2 | 信じられていた○○は幻想 | 「○○すれば安心、は幻想です」「正しいと思ってた○○、時代遅れ」 |
| 3 | ○○してるのに、うまくいかない理由 | 「努力してるのに成果が出ない。実は○○がズレてるだけ」 |
| 4 | あなたの常識は"いつ"作られた？ | 「その常識、いつ作られたか知ってますか？昔は正解、今は逆」 |
| 5 | ○○すればうまくいくは一部だけ | 「そのノウハウ、あなたに合ってないかも。○○タイプにしか向かない」 |
| 6 | 努力が報われない本当の理由 | 「努力が報われないのは○○のせいじゃない。戦う場所が違うから」 |
| 7 | 意外と知られてない落とし穴 | 「○○って実はうまくいかない人を量産してる」「知らないと損する真実」 |
| 8 | 成果が出る人は○○してない | 「結果を出す人は、むしろ○○を"やらない"」 |

> 「不」起点の作り方：ペルソナの不安/不満/不快を、①反対に言い換える ②よくある常識の逆を突く ③ありがちな行動の落とし穴を指摘、で企画化する。

## バズ10軸・心理トリガー早見表（補助に使う）

| 軸 | 効く理由 | 例 |
|---|---|---|
| 網羅性 | 全体像・完全解説 | 「初心者向け◯◯完全ガイド」 |
| まとめ系 | 特徴/NG/ランキング | 「やってはいけない習慣◯選」 |
| プロスペクト | 損失回避 | 「この食べ方で老け顔になります」 |
| 常識破壊 | 前提が崩れる驚き | 「水は情報を記憶している」 |
| 希少性 | 知られてない | 「日本で3人しか知らない◯◯」 |
| 権威性 | 実績・肩書き | 「医師が教える◯◯の真実」 |
| ザイガニック | 続きが気になる | 「この後、奇跡が起きます」 |
| カリギュラ | 禁止で見たくなる | 「絶対に真似しないでください」 |
| バンドワゴン | 乗り遅れ不安 | 「今話題！2025年ブーム」 |
| ニュース性 | 時事・流行 | 「ノーベル賞◯◯を解説」 |

**パワーワード**：完全版/永久保存版/禁断/必見/今すぐ/一撃/超有料級/無料/99%が知らない/実は/〜するだけ/方程式/ロードマップ/秘密。

## 企画審査5項目（全案セルフ採点・5中4以上で合格）

```
1 起点は「視聴者の今の痛み・恐怖・願望」か？（驚き情報スタートはFAIL）
2 タイトルを見て「これ、私のことだ」と思えるか？（自己投影性）
3 「過去の努力が報われなかった理由」を新視点で説明しているか？
4 メカニズムに固有名詞（チャンネル独自ワード）がついているか？
5 視聴者が動画内で「今すぐ確認・体験」できる要素があるか？（セルフチェック等）
```
**NG典型**：「なぜ○○は△△なのか」型（自分ごと化しない）／「知らなかった○○の事実」型（悩みと直結しない）／「○○する方法3選」型（汎用的すぎ）。

## 検索型 / ブラウジング型 戦略早見表

| | 検索・関連型（資産型・推奨first） | ブラウジング型（バズ型） |
|---|---|---|
| 流入 | 検索・関連動画 | ホーム（おすすめ） |
| 視聴者 | 悩み解決層（顕在） | 暇つぶし層（潜在） |
| 鍵 | キーワード選定・網羅性・回答精度 | CTR・サムネ・話題性・感情 |
| 資産性 | 長期（数年伸びる） | 短期（1ヶ月で収束しやすい） |
| 推奨 | 最初に取り組む | 知名度がついてから |

## 冒頭30秒フック構成テンプレート

```
1 テーマ＋衝撃結論「結論から言うと○○なんです」（※台本ではAI構文を避け言い換える）
2 天国と地獄：「これができると○○な未来」⇔「気づかないと○○な損」
3 希少性・権威性：「これは○○だけが知っている／私が△名を見てきて体系化した」
→ 30秒以内に視聴者の「自分ごと感」と「最後まで見る理由」を作る
```

## 専門用語 → 日常語 変換ルール

専門用語・概念名はタイトルに出さない。視聴者が「知る前から悩んでいる言葉」に変換する。

| NG（専門語） | OK（日常語） |
|---|---|
| 潜在意識の書き換え | 自分を変えたい人の共通点／習慣 |
| 量子理論 | 最先端科学が明かす"思考と現実"の関係 |
| エフィカシー | 自己肯定感と行動力の科学的な関係 |
| 共鳴場理論 | 脳が勝手に変わる習慣 |

# 台本タグ仕様

| タグ | 用途 | 画面構成 | 発音 |
|------|------|----------|------|
| `[見出し：◯◯]` | セクション見出し（左上バー）。次の見出しまで継続。スライド/Bロール中は非表示 | 左上ゴールド枠（アバター主画面・ピクチャー時のみ） | しない |
| `[スライドN]` | 重要ノウハウ・手順・図解・論理パート | スライド全画面、話者は右上ワイプ | しない |
| `[ピクチャーN]` | 感情的イメージ・具体物 | 話者メイン、左側に画像ポップアップ | しない |
| `[BロールN]` | シネマティックな情景・抽象概念の没入演出 | Bロール全画面、声のみ継続 | しない |
| `[カムリターン]` | 画像/スライド/Bロール表示を終了しアバターに戻す | カメラ映像に復帰 | しない |
| `<break time="0.8s" />` | 沈黙ポーズ | 沈黙（→編集でジェットカット） | 沈黙 |

## 沈黙マーカーのルール（厳守）
- **全ての文末**（「。」「！」「？」の直後）に `<break time="0.8s" />`。
- **全ての演出タグの直後**にも `<break time="0.8s" />`。
- 理由：編集はこの沈黙を編集点としてジェットカットする。間を作り「淀みなく喋り続ける不気味さ」を避けテンポを生む。

## 見出しタグの生成ルール
- 本題の各セクション冒頭に `[見出し：◯◯]`（章タイトル相当）。
- 短く・体言止め or 短いフレーズ（10〜16字）。看板コピーであって説明文ではない。
  - 良：「意識で物理宇宙を操作する」「なぜ、結果に差が出るのか？」
  - 悪：「ここからは○○について詳しく解説していきます」
- 1セクション1見出し。表示はアバター主画面・ピクチャー時のみ（スライド/Bロール全画面中は自動で非表示→カムリターンで再表示）。
- OP（フック・共感）にもつかみとして1つ置いてよい。
- `media_list.yaml` の `headings:` に `order` 順で全見出しを列挙し、台本タグと1対1対応。

## Bロールタグのルール（厳守・feedback_broll_tempo.md準拠）
1. タグ内に説明テキストを入れない（`[Bロール1]` のみ）。映像説明は `media_list.yaml` に書く。
2. `[BロールN]`〜`[カムリターン]` は原則「1文」で区切る（長くても2文・最長5秒）。AI動画モデル（Veo3等）の生成尺は数秒なので、**セリフ尺 < 生成動画尺**を担保する。長文を1つのBロールで流すと尺切れで静止画化する。
3. 3文以上の連続説明はBロールにしない。1文ごとに `[カムリターン]` でメインへ戻し、次の見せ場で `[Bロール2]`。区切りは必ず句読点（文末）。
4. クリップ尺の目安：1文≈3〜5秒。表示窓5秒超は `media_list.yaml` で `sub_clips:` に分割（各3〜4秒で発話セグメントと1対1）。
5. 1本でBロール挿入は3〜6箇所まで。使いすぎは「素材寄せ集め」感。

# 執筆ルール（CLBS文体）
- **会話調を徹底**：「〜だよね」「〜なんですよ」「実はさ…」。目の前の視聴者1人への対話として書く。スピーチ・アナウンス型は禁止。
- **yesセット**：OP・共感に「そうそう」と頷ける問いかけを3〜4連続。「〜ですよね」「〜ありませんか」「〜感じませんか」「〜ありますよね」。
- **AI構文の禁止**：
  - ×「結論から言います」「いきなり結論ですが」 → ○ 前置きせず本題（「いきなり答え言っちゃうんですけど、」）
  - ×「想像してみてください」 → ○「〜なこと、ありません？」
  - ×「例えるなら〜です」 → ○「これって要は〜と同じで、」
  - ×「〜について解説します」「今日は〜していきます」 → ○ 前置きなしで直入り
  - ×「〜していきましょう」「ご紹介させてください」 → ○「〜なんですよ」と切り込む
  - ×「ちょっと●●していいですか？」型の入りも禁止
- **AIっぽい表現の禁止**（feedback_ban_ai_expressions.md）：本文で `**`強調囲み・`——`/`ーー`の余韻ダッシュを使わない。
- **情報源の分離**：型（構成・リズム）はナレッジ参照可。中身（エピソード・固有名詞・データ）はユーザー入力／提出資料のみ。ナレッジ事例の中身は流用しない。

## heygen_script.txt の作り方
`script.txt` から機械的に:
- `[見出し：…]` `[スライドN]` `[ピクチャーN]` `[BロールN]` `[カムリターン]` の `[…]` タグを全て削除。
- `<break time="0.8s" />` は残す（HeyGenが沈黙として解釈）。
- タグ削除で `<break>` が連続したら1つに畳む。
- 結果はナレーション本文＋breakのみ。HeyGen（AI Studio・16:9・main cameraルック）に貼り付けて `video.mp4` を生成。

## media_list.yaml スキーマ

```yaml
video_info:
  title: "動画タイトル"
  channel: "チャンネル名"
  target: "ターゲット層"
  created: "YYYY-MM-DD"
  strategy: "ブラウジング型"   # or 検索・関連型

headings:                      # [見出し：◯◯] と1対1
  - order: 1
    text: "セクション見出し（10-16字）"

brolls:                        # [BロールN] と1対1
  - number: 1
    prompt: "15〜30語の英語プロンプト（cinematic 4K 等のスタイル指定込み）"
    keywords: "pexels検索キーワード（英語・半角スペース区切り）"
    duration: 4                # 3〜4秒推奨・上限5秒
```
