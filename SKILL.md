# clbs-video-script-pro

顔出しHeyGenアバター動画の台本を、後段の **clbs-youtube-edit**（最強版編集スキル）にそのまま投入できる形で執筆する。

## 位置づけ・出力

CLBS動画制作パイプラインの台本執筆スキル。`11_sns.md`（clbs-sns）のYouTube本編執筆ルールを継承しつつ、pro編集パイプライン向けに次を変更する。

- 沈黙マーカーは `<#0.8#>` ではなく **HeyGen SSML形式 `<break time="0.8s" />`**
- **見出しタグ `[見出し：◯◯]` を生成**（左上見出しバー用）
- 演出タグは**すべて無発音**前提（HeyGenには視覚タグを除いた版を読ませる）

**出力ファイル（プロジェクトフォルダ `videos/YYYYMMDD_<title>/`）:**

| ファイル | 内容 |
|---|---|
| `script.txt` | 演出タグ＋breakを全て含む完成台本（編集スキルの正） |
| `heygen_script.txt` | `[…]`視覚・見出しタグを除去し `<break>` のみ残した版（HeyGen貼り付け用） |
| `media_list.yaml` | 素材台帳（headings / brolls / pictures / slides / thumbnail） |

## タグ仕様

| タグ | 用途 | 画面構成 | 発音 |
|------|------|----------|------|
| `[見出し：◯◯]` | セクション見出し（左上バー）。次の見出しまで継続表示。ただしスライド/Bロール表示中は非表示 | 左上にゴールド枠の見出しバー（アバター主画面・ピクチャー表示中のみ） | しない |
| `[スライドN]` | 重要ノウハウのまとめ・手順・図解・論理パート | スライド全画面、話者は右上ワイプ | しない |
| `[ピクチャーN]` | 感情的なイメージ・具体物 | 話者メイン、左側に画像ポップアップ | しない |
| `[BロールN]` | シネマティックな情景・抽象概念の没入演出 | Bロール全画面、声のみ継続 | しない |
| `[カムリターン]` | 画像/スライド/Bロール表示を終了しアバターに戻す | カメラ映像に復帰 | しない |
| `<break time="0.8s" />` | 沈黙ポーズ | 沈黙（→編集でジェットカット） | 沈黙 |

### 沈黙マーカーのルール（厳守）

- **全ての文末**（「。」「！」「？」の直後）に `<break time="0.8s" />` を付与する。
- **全ての演出タグの直後**にも `<break time="0.8s" />` を付与する。
- 理由：pro編集はこの沈黙を編集点としてジェットカットする。間を作ることで「カメラ目線で淀みなく喋り続ける不気味な動画」を避け、テンポを生む。

### 見出しタグの生成ルール（新規）

- 本題の**各セクションの冒頭**に `[見出し：◯◯]` を置く（章タイトル相当）。
- テキストは**短く・体言止め or 短いフレーズ**（目安10〜16字）。説明文ではなく看板コピー。
  - 良い例：「意識で物理宇宙を操作する」「宇宙と繋がる量子脳」「なぜ、結果に差が出るのか？」
  - 悪い例：「ここからは量子脳がどう宇宙と繋がるのかについて詳しく解説していきます」（長すぎ・説明的）
- 次の `[見出し：…]` が出るまで表示が続くので、**1セクション1見出し**を基本にする。
- **表示されるのはアバター主画面とピクチャー表示中のみ**。`[スライドN]` `[BロールN]` の全画面表示中は見出しバーを**非表示**にし、`[カムリターン]` でアバターに戻ったら、その時点で有効な見出しを再表示する（編集スキル側で自動制御）。
- OP（フック・共感）にも、つかみとして1つ置いてよい。
- `media_list.yaml` の `headings:` に `order` 順で全見出しを列挙し、台本のタグと1対1で対応させる。

### Bロールタグのルール（厳守・feedback_broll_tempo.md準拠）

1. **タグ内に説明テキストを入れない**（`[Bロール1]` のみ）。映像説明は `media_list.yaml` に書く。
2. **`[BロールN]`〜`[カムリターン]` は原則「1文」で区切る**（長くても2文・最長5秒）。AI動画モデル（Veo3等）が生成できる尺は数秒なので、**セリフ尺は必ず生成動画尺より短く**する。長い説明を1つのBロールで流すと、動画が途中で尽きて静止画になったままセリフだけ続く事故になる。
3. **3文以上の連続説明はBロールにしない**。1文ごとに `[カムリターン]` でメイン動画へ戻し、次の見せ場で改めて `[Bロール2]` を置く。区切りは必ず句読点（文末）で行う。
4. クリップ尺の目安：1文≈3〜5秒。表示窓が5秒を超える場合は `media_list.yaml` で `sub_clips:` に分割（各3〜4秒で発話セグメントと1対1対応）。
5. 1本の動画でBロール挿入は**3〜6箇所**まで。使いすぎは「素材の寄せ集め」感が出る。

> 編集スキル(`clbs-youtube-edit`)側はBロール動画を無限ループ合成して静止を防ぐ安全策を持つが、ループの繋ぎ目が見えるので**台本側でセリフ<動画尺を担保するのが正**。

## 執筆ルール（CLBS文体・11_sns.md準拠）

- **会話調を徹底**：「〜だよね」「〜なんですよ」「実はさ…」など、目の前の視聴者1人への直接の会話として書く。スピーチ・アナウンス型は禁止。
- **yesセット**：OP・共感パートに、視聴者が「そうそう」と頷ける問いかけを3〜4連続で入れてから核心へ。問いかけ形は「〜ですよね」「〜ありませんか」「〜感じませんか」「〜ありますよね」。
- **AI構文の禁止**：
  - ×「結論から言います」「いきなり結論ですが」 → ○ 前置きせず本題に入る
  - ×「想像してみてください」 → ○「〜なこと、ありません？」
  - ×「例えるなら〜です」 → ○「これって要は〜と同じで、」
  - ×「〜について解説します」「今日は〜していきます」 → ○ 前置きなしで直入り
  - ×「〜していきましょう」「ご紹介させてください」（誘導・許可求め） → ○「〜なんですよ」と切り込む
  - ×「ちょっと●●していいですか？」型の入り方も禁止
- **AIっぽい表現の禁止**（feedback_ban_ai_expressions.md）：本文で `**`強調囲み・`——`/`ーー`の余韻ダッシュを使わない。
- **情報源の分離**：型（構成・リズム）はナレッジを参照してよいが、中身（エピソード・固有名詞・データ）はユーザー入力／提出資料のみを使う。ナレッジ事例の中身を流用しない。

## ワークフロー

ヒアリング済み情報があればスキップしてよい。承認ゲートを各フェーズで挟み、一度に全部出力しない。

1. **ヒアリング**：テーマ／ターゲットの悩み／伝えたいノウハウ・独自エピソード／CTA（誘導先）／話し方トーン。
2. **構成案**：OP（フック・ベネフィット・権威性）→共感（yesセット）→解決策（未来提示＋前半CTA）→本題（見出し3〜4つ）→まとめ（最終CTA）。本文は書かず骨子のみ。→ 承認。
3. **本文執筆**：1パートずつ。各パート冒頭に `[見出し：◯◯]`、論理パートに `[スライドN]`、感情・具体物に `[ピクチャーN]`、没入演出に `[BロールN]`＋`[カムリターン]` を配置。全文末・全タグ直後に `<break time="0.8s" />`。→ パートごとに承認。
4. **素材台帳ほか出力**：`script.txt`（完成版）→ `heygen_script.txt`（タグ除去版を生成）→ `media_list.yaml`（headings/brolls/pictures/slides/thumbnail）。

### heygen_script.txt の作り方

`script.txt` から次を機械的に行う：
- `[見出し：…]` `[スライドN]` `[ピクチャーN]` `[BロールN]` `[カムリターン]` などの `[…]` タグを**全て削除**。
- `<break time="0.8s" />` は**残す**（HeyGenが沈黙として解釈する）。
- タグ削除で `<break>` が連続したら1つに畳む。
- 結果はナレーション本文＋breakのみ。これをHeyGen（AI Studio・16:9・main cameraルック）に貼り付けて `video.mp4` を生成する。

## media_list.yaml スキーマ（pro版）

```yaml
video_info:
  title: "動画タイトル"
  channel: "チャンネル名"
  target: "ターゲット層"
  created: "YYYY-MM-DD"
  strategy: "ブラウジング型"   # or 検索・関連型

headings:                      # [見出し：◯◯] と1対1（pro新設）
  - order: 1
    text: "セクション見出し（10-16字）"

brolls:                        # [BロールN] と1対1
  - number: 1
    prompt: "15〜30語の英語プロンプト（cinematic 4K 等のスタイル指定込み）"
    keywords: "pexels検索キーワード（英語・半角スペース区切り）"
    duration: 4                # 3〜4秒推奨・上限5秒
    aspect_ratio: "16:9"
    source: pexels             # pexels / seedance / auto
  # 表示窓>5秒は sub_clips に分割
  - number: 2
    sub_clips:
      - { duration: 4, prompt: "...", keywords: "..." }
      - { duration: 4, prompt: "...", keywords: "..." }
    aspect_ratio: "16:9"
    source: auto

pictures:                      # [ピクチャーN] と1対1
  - number: 1
    prompt: "English prompt（具体的で視覚的に）"
    aspect: "1:1"              # 左ポップアップ用

slides:                        # [スライドN] と1対1
  - number: 1
    title: "スライド日本語タイトル"
    subtitle: "サブタイトル（任意）"
    description: "載せる内容要素"
    layout: "レイアウト指示"
    tone: "背景色・デザイントーン"

thumbnail:
  title_text: "サムネ文字（10-15字）"
  visual_concept: "ビジュアルコンセプト"
  color_scheme: "配色"
  aspect: "16:9"
```

**番号整合性（厳守）:** `script.txt` のタグ番号と `media_list.yaml` の `number:`／`headings.order` を必ず1対1対応させる。タグの無い項目は削除、タグがあるのに項目が無い状態は禁止。

## 完了チェックリスト

```
■ 台本（script.txt）
□ 各セクション冒頭に [見出し：◯◯]（短い看板コピー）が置かれている
□ [スライド]/[ピクチャー]/[Bロール]/[カムリターン] が用途通り配置されている
□ Bロールタグ内に説明テキストが無い（単体タグ）
□ 全Bロール区間に2〜3文以上のセリフがある／挿入数3〜6箇所
□ 全文末・全タグ直後に <break time="0.8s" /> が付いている
□ yesセット（OP・共感に3〜4連続）／会話調／AI構文なし／** ・—— 不使用

■ HeyGen入力（heygen_script.txt）
□ [...] 視覚・見出しタグが全て除去され、break のみ残っている

■ 素材台帳（media_list.yaml）
□ headings/brolls/pictures/slides が台本タグと1対1対応
□ 表示窓5秒超のBロールは sub_clips で分割済み
□ video_info / thumbnail が記載済み
```
